# Support Policy

This document describes the default support policy for licensed DelphiMTKClient buyers.

## 1. Standard Support Period

Licensed buyers receive **one year of support** from the purchase date, unless a separate written agreement says otherwise.

During this period, the buyer may contact Alephgsm for help related to:

- Project setup.
- Compile issues.
- Debugging guidance.
- Runtime file placement.
- Driver preparation.
- Device test preparation.
- Minor update questions.
- Bug reports.
- Workflow clarification.

Contact:

https://t.me/GsmCoder

## 2. Minor Updates and Bug Fixes

For one year after purchase, minor updates and bug-fix updates are provided free of charge when available.

Examples of minor updates:

- Bug fixes.
- Stability fixes.
- Small compatibility improvements.
- Minor UI corrections.
- Small workflow improvements.
- Small protocol fixes that do not introduce a major new paid feature.

## 3. Major Features

If a major new feature is added after the buyer's purchase, the buyer does **not** need to buy the full project again.

Instead, the buyer only pays the value of the new major feature if they want that feature.

Examples of major features may include:

- A new major protocol module.
- A new brand-specific workflow.
- A new advanced repair or patch module.
- A new major filesystem or storage workflow.
- A new major security / authentication path.
- A large new commercial module.

Major feature pricing depends on the feature scope.

## 4. Support Contact

If the buyer has a problem, they can contact Alephgsm support:

https://t.me/GsmCoder

When requesting support, provide:

- Purchase name or license reference.
- Delphi / RAD Studio version.
- Windows version.
- Build target and configuration.
- Error message or screenshot.
- Operation log from the application.
- Device chipset / model if related to device testing.
- Files or paths involved, if relevant.

## 5. What Support Covers

Support generally covers:

- Helping the buyer compile the project.
- Explaining required folders and runtime files.
- Helping identify missing drivers.
- Reviewing logs for obvious setup or workflow problems.
- Clarifying how a feature is intended to work.
- Providing available minor updates and bug fixes.

## 6. What Support Does Not Cover by Default

Standard support does not automatically include:

- Building a full custom product for the buyer.
- Adding private features for free.
- Supporting unauthorized shared copies.
- Supporting modified code that was heavily changed by the buyer without review.
- Reverse engineering a new unsupported device model as free support.
- Providing files, loaders or credentials that are not part of the purchased package.
- Supporting buyers who violated the license terms.

Custom development can be discussed separately.

## 7. License Compliance

Support is available only for legitimate licensed buyers.

If the source code was resold, exchanged, gifted, published, leaked or shared without permission, support may be refused.
