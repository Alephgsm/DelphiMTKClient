# Buyer Guide

DelphiMTKClient is a commercial Delphi source code product for MediaTek / MTK protocol development.

This guide explains who the project is for, what buyers should expect and how the project can be used as a development foundation.

## Who Should Buy

DelphiMTKClient is suitable for:

- Delphi developers who need MTK protocol source code.
- Software owners building a private MTK service product.
- Companies that need an internal MediaTek maintenance tool.
- Research teams studying MTK protocol behavior.
- Developers who want to build branded Windows software.
- Teams that need custom workflows, licensing and private logic.

## Who Should Not Buy

This project is not designed for regular end users who only want a compiled repair tool.

Buyers should have at least one developer who can read and maintain Delphi code.

## What You Can Build

With the commercial source package, developers can build:

- A private MTK flashing tool.
- A branded service software product.
- A device maintenance platform.
- A model-specific repair workflow.
- A research and protocol analysis application.
- A Windows Delphi product with custom licensing.
- Internal tools for a service company or lab.

## Development Requirements

Typical requirements:

- Windows development environment.
- Delphi / RAD Studio knowledge.
- Understanding of MTK workflows.
- Ability to test with real devices.
- Knowledge of device modes, DA files and firmware packages.

## Important Technical Notes

MediaTek behavior is not identical across all devices.

Feature availability depends on:

- Chipset family.
- Boot mode.
- Selected Download Agent.
- Device security configuration.
- Storage type.
- Firmware package format.
- Protocol mode: LEGACY, XFLASH or XML.
- Brand-specific implementation details.

## Before Purchase

Recommended questions:

- Which workflow do you need most: flash, partition, file management, IMEI, RPMB, patch, research?
- Do you need source code for customization or only a ready tool?
- Do you have Delphi development knowledge?
- Do you need branding and licensing integration?
- Do you need model-specific development after purchase?

## Support Summary

Licensed buyers receive one year of minor updates and bug-fix support by default, unless a separate written agreement says otherwise.

If a major new feature is added later, the buyer does not need to purchase the full project again. The buyer only pays the value of the new major feature if they want that feature.

If the buyer has a problem, they can contact Alephgsm support and provide build details, logs, screenshots and device information where relevant.
