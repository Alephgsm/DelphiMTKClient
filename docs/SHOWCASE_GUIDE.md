# GitHub Showcase Guide

Use this guide when publishing the DelphiMTKClient public GitHub repository.

The goal is to make the repository look professional while protecting the commercial source code.

## Recommended Repository Name

Recommended:

```text
DelphiMTKClient
```

Alternative names:

```text
DelphiMTKClient-Source-Code
Delphi-MTK-Protocol-Source-Code
MTK-Protocol-Delphi-Source
```

## Screenshots To Add

Add polished screenshots, but avoid exposing sensitive implementation details.

Recommended screenshots:

- Main UI dashboard.
- Flash tab.
- Partition Management tab.
- File Management partition browser.
- Source code structure in the IDE.
- Protocol module folders.
- Log output after a successful device initialization.
- Feature tabs showing available workflows.

Avoid screenshots showing:

- Private payload implementation.
- Full exploit code.
- Authentication material.
- DA or loader internals that should remain private.
- Customer-specific logic.
- Private keys or API endpoints.

## Professional Tips

- Use a dark 16:9 banner image at the top of the README.
- Add GitHub badges for platform, language and protocol support.
- Keep the repository public, but do not include the real source package.
- Use `NOTICE.md` to clarify that this is a product showcase.
- Use `SECURITY.md` to avoid public posting of sensitive material.
- Pin the repository on the GitHub profile.
- Add the YouTube demo links to the README after uploading videos.
- Use the same branding across GitHub, YouTube, Telegram and the website.
